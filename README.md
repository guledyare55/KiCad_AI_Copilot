# KiCad AI Copilot

> **EDA Design Generator** — Convert natural language electronics requirements into KiCad-ready PCB designs.

---

## 🚀 Quick Start

### 1. Open the Web Interface
Open `index.html` in any modern browser (Chrome, Edge, Firefox).

```
KiCad_AI_Copilot/
  index.html        ← Open this
  style.css
  app.js
  engine.js
  kicad_generator.py
```

### 2. Generate a Design
1. Enter your project name and description
2. Select board size, power source, and cost target
3. (Optional) List specific components
4. Click **Generate KiCad Design**

### 3. Export & Convert to KiCad Files
1. Click **Export JSON** to download the design JSON
2. Run the Python generator:

```bash
python kicad_generator.py --input my_design.json --output ./my_project/
```

## ⚡ What Gets Auto-Generated

| Category | What's Added |
|----------|-------------|
| **Decoupling** | 100nF cap per VCC pin + bulk bypass cap |
| **MCU Reset** | 10kΩ pull-up on /RESET |
| **Programming** | SWD (1.27mm 4-pin) or UART header |
| **Buttons** | 10kΩ pull-up + 100Ω debounce series resistor |
| **LEDs** | 330Ω current limiting resistor |
| **Battery** | Schottky reverse-polarity diode + 4.7µF filter cap |
| **USB-C** | 5.1kΩ CC1/CC2 pull-downs + ferrite bead EMI filter |
| **RF/BLE/LoRa** | π-network antenna matching (1.5pF / 2.2nH / 1pF) |
| **I2C Sensors** | 4.7kΩ SDA + SCL pull-ups |
| **LDO (USB)** | AP2112K-3.3 + 1µF in + 10µF out caps |

## 🔌 Electronics Design Rules Enforced

1. **Power**: Every IC gets 100nF decoupling + bulk bypass cap
2. **MCU**: Reset pull-up + programming header always added
3. **Buttons**: Pull-up + debounce series resistor per button
4. **LEDs**: Current limiting resistor calculated at ~10mA
5. **RF**: Antenna keepout, 50Ω trace, π-matching network
6. **USB**: CC pull-downs, VBUS ferrite bead, decoupling
7. **I2C**: Bus pull-ups added when sensor detected
8. **Battery**: Schottky protection + filter capacitor

## ⚠️ Post-Generation Steps in KiCad

1. **Open** `project.kicad_pro` in KiCad v7+
2. **Schematic**: Wire components using net labels, run ERC
3. **PCB**: Update PCB from schematic, use auto-placer as starting point
4. **Placement**: Follow `placement_hints.json` for critical components
5. **RF**: Pay strict attention to RF rules — antenna keepout, 50Ω traces
6. **DRC**: Run Design Rules Check before sending to fab
7. **Gerbers**: Export to `gerbers/` folder using included plot settings
